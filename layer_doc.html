Layer.pm

Used to help with scripts associated with Genesis on actions related to the layer. In this context "Layer" pertains to the matrix, and functions relating to such.

SYNOPSIS

     ##################                          ##################
     ################## EXAMPLES OF SIMPLE USAGE ##################
     ##################                          ##################
     use Layer;
     use Job;
     use Genesis;
     my $f = new Genesis;
     my $job = Job->new(PTX => $ENV{JOB}, GENESIS => $f);
     my $layer = Layer->new(NAME => '1a', JOB_MOD=> $job, STEP => 'pcb');
     
     my $side = $layer->getSide();
     print "$side\n"; #top
     
     #make the layer mixed
     $layer->setType("mixed");
     
     #make it misc
     $layer->setContext("misc");
     
     #make a new layer object
     $layer = Layer->new(NAME => 'd1', JOB_MOD=> $job, STEP => 'pcb');
     
     my $start = $layer->getStartName();
     print "$start\n"; #prints a layer name

METHODS

--Initializer--

new

new(NAME => 'layer_name',JOB_MOD=>$job_obj,STEP=>'step_name')

Creates a new layer object.

--Getters--

getEndName

getEndName()

Returns the end name of the layer. If the layer type is either drill or rout it will be the name of another layer. Otherwise, it will be an empty string.

getStartName

getStartName()

Returns the start name of the layer. If the layer type is either drill or rout it will be the name of another layer. Otherwise, it will be an empty string.

getPolarity

getPolarity()

Returns the polarity of the layer; either "positive" or "negative".

getType

getType()

Returns the layer type; either "signal", "mixed", "power_ground", "solder_mask", "silk_screen", "solder_paste", "drill", "rout" or "document".

getRow

getRow()

Returns the layer's row number in the matrix. Indexing starts at 1; so the first layer in the matrix is row 1.

getContext

getContext()

Returns the layer context; either "board" or "misc".

getFoilSide

getFoilSide()

Returns the layer foil side; either "none", "top" or "bottom". Only copper layers will be "top" or "bottom" all others will be "none".

getSheetSide

getSheetSide()

Returns the layer sheet side; either "none", "top" or "bottom". Only copper layers will be "top" or "bottom" all others will be "none".

getSide

getSide()

Returns the layer side; either "top", "inner", "bottom" or "none". Rout, drill, and document layers will always be "none"

getName

getName()

Returns the name of the layer.

getStepName

getStepName()

Returns the step name associated with this layer.

getDisplayNumber

getDisplayNumber()

Returns a pair ($index, $number) where $index is the display number, and $number is the number of displayed layers. If the layer is not displayed $index will be -1.

--Setters--

setContext

setContext("context")

Takes one parameter; it can be either board or misc. Sets the layer context to the given parameter. It will return 1 if the action was successful, or 0 if it was unsuccessful.

setType

setType("type")

Takes one parameter; it can be either signal, mixed, power_ground, solder_mask, silk_screen, solder_paste, drill, rout or document. Sets the layer type to the given parameter. It will return 1 if the action was successful, or 0 if it was unsuccessful.

setPolarity

setPolarity(polarity)

Takes one parameter; either positive or negative. Sets the layer polarity to the give parameter. Returns 1 if the action was successful, or 0 if it was unsuccessful.

--Behaviors--

affect

affect()

Affects the layer in the graphic editor.

unaffect

unaffect()

Unaffects the layer in the graphic editor.

display

display(int)

Takes an optional parameter int; which can be an integer between 1 and 4 inclusive. It will display the layer in the graphic editor. If the parameter is not given it will default to the next display number, ie if there are already layers displayed it will be the next in the display order. The display order determines the display color.

displayOff

displayOff()

Turns off the display in the graphic editor. If the layer is not displayed to begin with then no action will be taken.

debugOn

debugOn()

Turns on helpful print statements for debugging. The default is off.

debugOff

debugOff()

This is the default behavior upon initialization. It will not print debug statements.
